# 2026-04-21 和弦分析与全曲预览收口

## 本轮目标

- 继续参考 `ChordMiniApp` 的输出组织方式，收紧当前 Android 原生和弦分析链。
- 解决“全曲分析能跑完，但下方预览内容不够、和弦仍不稳定”的问题。

## 本轮改动

### 1. 原生和弦分析切段与后处理

文件：`android/app/src/main/java/cn/toside/music/mobile/mixer/MixerModule.java`

- 将和弦基础切段从固定 `2000ms` 下调到 `700ms`，先细粒度采样，再做后处理合并。
- 调整模板打分：
  - 改为按归一化 pitch-class 能量计算 chord-tone / non-chord-tone 比例。
  - 强化根音、三音、五音、七音支持度。
  - 强化调内和弦优先级与常见级进过渡奖励。
- 新增后处理：
  - 合并相邻相同和弦。
  - 吸收短暂桥接段。
  - 清洗低置信度的短抖动段。

### 2. 二级分析页扩展为可展开的全曲预览

文件：`src/screens/PlayDetail/components/KeyInfoBtn.tsx`

- 歌词对齐预览不再固定只看前 `28` 行。
- 新增：
  - 全曲和弦分段展开/收起
  - 全曲歌词对齐展开/收起
- 针对不常用写法做展示层修正：
  - 将 `Abm/Dbm/Gbm` 这类不利于当前弹唱阅读的记名转成更常见的等音写法后再显示。

## 参考说明

- 在线参考仓：`https://github.com/ptnghia-j/ChordMiniApp`
- 结论：
  - 该仓的后端以 Python 服务封装为主，核心价值更偏“统一输出结构 + 模型服务接入层”。
  - 现有 RN Android 本地链路不适合直接搬它的模型推理代码，因此本轮主要借鉴“结果组织方式”和“让和弦输出更稳定”的方向，而非机械复制服务实现。
