# v0.7 启动默认音源与歌词/KTV 测试说明

生成时间：2026-04-17 10:23:12（UTC+08:00）

## 本轮目标

1. 删除启动后的许可协议确认入口。
2. 将默认自定义音源内置进应用并在首次迁移时自动启用。
3. 启动后主动申请存储权限，并初始化下载目录。
4. 强化下载歌词同步，避免只下载音频时无提示丢歌词。
5. 参考 `C:\Users\Administrator\Desktop\lx二开\测试用曲目` 增加 KTV 伴奏识别测试。

## 架构拆分

- 启动链路：
  `src/core/init/index.ts`
  负责移除协议拦截、首次启动权限申请、默认下载目录初始化。
- 默认音源链路：
  `src/config/defaultUserApiScript.ts`
  `src/core/init/userApi/index.ts`
  将 Windows 本地脚本打进应用包，在初始化时自动导入并仅做一次默认启用迁移。
- 下载歌词链路：
  `src/core/download/index.ts`
  对歌词做缓存/刷新两轮拉取，并把扩展歌词内容统一写入 `.lrc`。
- KTV 测试链路：
  `src/utils/ktv.js`
  `src/utils/localMediaMetadata.ts`
  `tests/test_ktv_fixture.js`
  负责识别同名伴奏变体、过滤本地列表重复展示，并用夹具目录做回归验证。

## 已完成改动

- 启动时不再弹许可协议确认。
- 关于页不再保留“许可协议”确认入口，只保留在线协议链接。
- 默认内置音源改为随应用打包，首次迁移时会自动导入并切换到 `[独家音源]`。
- 启动后会主动弹存储权限请求；若允许，则自动将下载目录设置到默认下载目录并创建目录。
- 下载完成后会优先写入完整歌词；如果歌词接口无结果，任务状态会明确显示 `Completed (lyric unavailable)`，不再静默成功。
- 本地媒体扫描会把 `htdemucs_no_vocals`、`htdemucs_minus_other`、`htdemucs_vocals` 等同名 KTV 变体折叠到同一首歌下，避免列表里重复显示多个拆轨文件。
