# v1.5 目录返回 / KTV 归组 / 本地去重修复

时间：2026-04-17 14:36:50 +08:00

## 本次目标

修复以下三个用户现场问题：

1. 添加音乐时，目录选择器无法返回上一层，提示“无权限”。
2. 播放页点击 KTV，提示没有相关音轨。
3. 本地音乐仍然出现大量重复展示，用户要求按“文件名前几位 + 同文件时间”合并展示。

## 架构拆分

- 目录访问链：`ChoosePath -> List -> readDir`
- KTV 文件识别链：`ktv.js -> core/ktv -> PlayDetail/KtvBtn`
- 本地扫描展示链：`localMediaMetadata -> LocalMusic -> MusicList`

## 实际改动

### 1. 目录返回上级修复

文件：`src/components/common/ChoosePath/List.tsx`

- 原逻辑只依赖 `parentDirInfo` 记录“从哪个目录点进来”的父子关系。
- 当初始目录是直接打开、或某些路径没有被缓存时，点击返回上一级就会直接报 `Permission denied`。
- 现改为：
  - 打开初始目录时先推导并记录父目录；
  - 返回上级时优先用缓存父目录；
  - 缓存不存在时，根据当前路径直接推导父级路径；
  - 到达顶层时提示“已经是顶层目录”，不再误报无权限。

### 2. KTV 音轨归组修复

文件：

- `src/utils/ktv.js`
- `src/core/ktv/index.ts`

改动点：

- 增强 KTV 后缀识别，支持：
  - `vocal / vocals`
  - `bgm`
  - `others`
  - `bass`
  - `drum / drums`
  - `htdemucs_*`
  - 带空格、下划线、括号、中括号、中文括号等尾缀形式
- 归组键不再只看完整基础名，加入“规范化前缀 + 文件时间”作为合并依据，贴合用户现有文件规则。
- 播放页 KTV 弹窗标签优先展示实际差异后缀，例如：
  - `vocal`
  - `bgm`
  - `others`
  - `drums`
- 主文件固定显示为“主文件”。

### 3. 本地音乐重复展示修复

文件：

- `src/utils/ktv.js`

改动点：

- 本地扫描层继续调用 `filterKtvDisplayFiles()`，但其内部归组规则已升级为“前缀 + 时间”。
- 同一组只保留一个代表文件用于列表展示，其他 KTV 衍生轨不再单独出现在本地音乐列表。
- 若目录中只有衍生轨没有主文件，则按优先级回退展示一个可播放文件。
