# KTV 切换运行时异常修复说明

时间：2026-04-17 16:00:59 +08:00

## 问题现象

播放中的 KTV 音轨切换会稳定报错：

- `switchKtvVariant failed`
- `TypeError: undefined is not a function`

## 定位结果

通过 `adb logcat --pid` 抓取运行时日志，并结合 Hermes sourcemap 反解，异常偏移定位到：

- `src/core/ktv/index.ts:246`

对应问题点是 `switchKtvVariant()` 内对以下调用结果直接链式 `.catch()`：

- `getLyricInfo({ musicInfo: nextMusicInfo }).catch(...)`
- `getPicPath({ musicInfo: nextMusicInfo, ... }).catch(...)`

该写法在返回值不是标准 Promise，或导出在运行时链路中临时未就绪时，会触发 `undefined is not a function`。

## 修复内容

已将以下位置改为显式 `try/catch` 包装：

- 读取当前播放进度
- 读取拍点分析结果
- 预取歌词与封面元数据
- 调用 `startResourceTransition()`

这样即使某一步返回值形态异常，也只会降级，不会中断整个 KTV 切换流程。
