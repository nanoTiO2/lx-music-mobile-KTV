# 歌词舞台退出修复_提词被控端入口说明

时间：2026-04-21 13:25 +08:00

## 本轮目标

继续推进当前可测试交付，处理两个直接影响使用的问题：

- 修复歌词舞台退出后偶发残留全屏沉浸式状态
- 在设置页提供“被控端”入口，落独立简化 UI，并支持退出

## 本轮改动

### 1. 歌词舞台退出修复

已更新：

- `src/screens/LyricStage/index.tsx`

本次处理方式：

- 把系统 UI 恢复收口为显式 `restoreStageSystemUi`
- 点击返回按钮时，不再只依赖组件卸载清理
- 在 `componentDidDisappear` 时再次兜底恢复

修复目标：

- 从播放页进入歌词舞台，再返回播放页时
- 不再残留全屏沉浸式状态
- 减少播放页与系统状态栏重叠

### 2. 设置页“被控端”入口

已新增：

- `src/screens/Home/Views/Setting/settings/Other/PromptReceiverEntry.tsx`

已接入：

- `src/screens/Home/Views/Setting/settings/Other/index.tsx`

当前入口位置：

- 设置
- 其他
- 提词被控端

### 3. 独立被控端屏

已新增：

- `src/screens/PromptReceiver/index.tsx`

当前能力：

- 独立全屏简化 UI
- 显示当前句 / 下一句
- 显示会话来源、链路、模式、状态
- 提供显式退出按钮
- 与当前 `PromptSession` 共享同一份本地会话状态
- 支持填写 `WebSocket` 地址与会话号接入远端控制快照

当前定位：

- 先作为“被控端原型屏”
- 后续接入 WebSocket / WebRTC / 蓝牙后，可直接承接真实被控态

### 4. 导航更新

已更新：

- `src/navigation/screenNames.ts`
- `src/navigation/navigation.ts`
- `src/navigation/registerScreens.tsx`
- `src/screens/index.ts`

### 5. WebSocket 客户端接入

已更新：

- `src/shared/prompt/session.ts`
- `src/screens/PromptControl/index.tsx`
- `src/screens/PromptReceiver/index.tsx`

当前行为：

- 控制端可填写 `WebSocket` 地址与 `sessionId`
- 控制端连接后会发送整份提词快照
- 被控端可使用同一地址和会话号接入
- 断开后自动回退到本地 `loopback`

当前定位：

- 这是提词链路的移动端客户端接入层
- 当前仓未内置 WebSocket 服务端
- 若要跨设备联动，仍需要一个可转发提词消息的 WebSocket 服务
