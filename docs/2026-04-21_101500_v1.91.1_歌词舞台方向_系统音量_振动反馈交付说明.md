# v1.91.1 歌词舞台方向 / 系统音量 / 振动反馈交付说明

## 本次目标
- 歌词舞台默认跟随屏幕方向，手动后锁定横屏，并可切回自动跟随。
- 主要交互增加更有手感的振动反馈。
- 增加系统音量控制入口，适配无实体音量键设备。
- 调号分析弹窗增加升降调、播放速率快捷按钮。
- 重新构建、安装到测试手机，并推送代码与 tag。

## 已完成改动
- 歌词舞台方向逻辑改为两态：
  - 默认 `auto`，跟随当前屏幕方向。
  - 点击后切到 `landscape` 锁定横屏，再点一次恢复跟随。
- 歌词舞台圆盘面板新增系统音量调节：
  - `-8%`
  - `50%`
  - `+8%`
  - 滑杆直调系统媒体音量。
- 播放详情页在调号分析按钮旁新增 `音量` 按钮：
  - 横屏页头可调。
  - 竖屏播放页工具区可调。
- Android 原生 `UtilsModule` 新增能力：
  - `getSystemVolume`
  - `setSystemVolume`
  - `adjustSystemVolume`
  - `playHaptic`
- 新增统一振动工具 `src/utils/haptics.ts`，并接入以下环节：
  - 播放 / 暂停
  - 上一首 / 下一首
  - 下载成功
  - 调号分析成功
  - 歌词舞台圆盘呼出
  - 歌词舞台拖动滑杆
- 调号分析弹窗新增快捷按钮：
  - 升降调 `-1 / 复位 / +1`
  - 播放速率 `-0.1 / 复位 / +0.1`
- 顺手收掉了 `src/store/player/action.ts` 中 `tempPlayList` 的 `listId` 类型不稳定问题，避免本轮提交继续带着隐性报错。
