## 实际改动

### 1. htdemucs 文件规则改为精确尾缀匹配

文件：`src/utils/ktv.js`

- 不再把 `htdemucs_minus_vocals` 错误识别成 `vocals`
- 增加精确尾缀优先级：
  - `htdemucs_minus_vocals`
  - `htdemucs_minus_other`
  - `htdemucs_vocals`
  - 以及普通 `minus_vocals / minus_other / vocals / bgm / others / bass / drums`
- 本地列表过滤后，仅保留主文件 `mp3`
- KTV 衍生轨不再在本地音乐列表单独展示

### 2. KTV 弹窗展示真实文件尾缀

文件：`src/core/ktv/index.ts`

- 主文件固定显示为 `主文件`
- 其他音轨优先显示真实尾缀，不再翻译为 `bgm/vocal`
- 对你这类文件名，弹窗将直接显示：
  - `htdemucs_minus_other`
  - `htdemucs_minus_vocals`
  - `htdemucs_vocals`

### 3. 切换对齐到下一拍

文件：`src/core/ktv/index.ts`

- 播放中点击 KTV 切换时，不立即切
- 先读取当前播放位置
- 基于 beat grid 计算下一拍时间点
- 到下一拍再执行切换
- 歌词与封面提前预取，切换后继续沿用主体歌词文件

说明：

- 当前这版实现的是“下一拍切换调度”
- 这样可以保证切换时机和拍点对齐
- 若后续你现场试听仍觉得切换瞬间有轻微断感，再继续把 native mixer 热切链做成默认路径
