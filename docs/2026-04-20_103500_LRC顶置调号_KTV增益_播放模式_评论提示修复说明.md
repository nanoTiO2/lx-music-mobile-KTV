# 2026-04-20 LRC 顶置调号、KTV 增益、播放模式与评论提示修复说明

## 本轮目标

- 识别调号后，将调号与分析时间写到 LRC 文件最顶部
- 让伴奏增益对所有非主音频分轨都生效
- 调整歌词舞台 `满屏 / 提词器` 模式视觉，保持与 `三行` 模式同一套审美
- 修复播放模式按钮，只保留并稳定切换：
  - 列表循环
  - 列表顺序
  - 单曲循环
- 本地音乐评论页改为灰色提醒态，明确提示无法获取评论

## 模块拆分

- `src/shared/musicProfileTag.ts`
  - 负责 LRC 分析标签生成与写入顺序
- `src/plugins/player/utils.ts`
  - 负责播放器与 mixer 的音量、KTV 增益重下发
- `src/screens/LyricStage/index.tsx`
  - 负责歌词舞台的三种展示模式和低耗模式视觉
- `src/screens/PlayDetail/*/PlayModeBtn.tsx`
  - 负责播放模式按钮的实际切换顺序
- `src/screens/Comment/index.tsx`
  - 负责评论页的本地歌曲提示态

## 关键改动

### 1. LRC 顶部写入调号与时间

- 写回时将以下标签固定插入到 LRC 最顶部：
  - `key`
  - `bpm`
  - `beat`
  - `analysis_time`
  - `lx_music_profile`
- `analysis_time` 改成更直接可读的时间字符串

### 2. KTV 非主音频增益

- mixer 激活时，不再只依赖分轨 gain
- 现在会把非主音轨增益同步体现在 mixer 输出音量上
- 目标是让所有非 `main` 变体都能明显感知到增益变化

### 3. 满屏 / 提词器模式视觉

- active 行增加卡片高亮、边框与阴影
- `teleprompter` 改为左对齐阅读感布局
- `full` 模式增加更明显的当前行承载块
- 文本增加 `capitalize`，英文歌词首字母可自动大写
- 低耗模式下保留最少量的 active 行动画感，其余行继续走低成本渲染

### 4. 播放模式按钮

- 横屏与竖屏按钮统一只切换 3 种常用模式：
  - 列表循环
  - 列表顺序
  - 单曲循环
- 不再混入随机和禁用，减少误切和状态混乱

### 5. 本地评论提醒

- 本地音乐评论页改为灰色提示卡片
- 明确提示当前歌曲无法获取评论
