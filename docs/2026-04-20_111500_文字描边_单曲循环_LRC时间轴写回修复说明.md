# 2026-04-20 文字描边、单曲循环、LRC 时间轴写回修复说明

## 本轮目标

- 删除歌词卡片描边，改成真正的文字描边效果，并提供独立开关
- 提高 KTV 非主音轨增益上限与滑块范围
- 修复单曲循环在播放结束后停住的问题
- 把本地评论页彻底改成灰色提醒态
- 将调号与分析时间写成 `0s~1s` 内可见的 LRC 时间轴行，并避开已有起始歌词

## 关键修复

### 1. 文字描边

- 新增设置项：
  - `lyricStage.textStroke`
- 歌词舞台控制面板新增按钮：
  - `描边:开 / 描边:关`
- 删除上一轮的卡片边框高亮，改为文字级阴影描边
- `full / teleprompter / threeLine` 统一回到居中对齐

### 2. KTV 增益

- 设置页滑块上限从 `180%` 提高到 `320%`
- mixer 输出音量单独允许高于 `1.0`
- 非主音轨 gain 现在不仅作用在轨道 gain，也会叠加到 mixer 输出音量

### 3. 单曲循环

- 播放结束事件新增应用层兜底
- 当播放模式为 `singleLoop` 时，结束后会直接从头重新播放当前曲目，而不是落到停止/暂停态

### 4. 本地评论灰色态

- 评论页 Header 也支持灰色背景
- 本地评论分支整体包裹在灰色页面中，不再只是一张中间卡片

### 5. LRC 时间轴写回

- 继续保留顶部元标签：
  - `key`
  - `bpm`
  - `beat`
  - `analysis_time`
  - `lx_music_profile`
- 额外写入两行可显示的 LRC 时间轴歌词：
  - `♪ 调号：...`
  - `♪ 分析时间：...`
- 时间优先放在 `0ms ~ 980ms`
- 如果起始段已有歌词，则自动选择未冲突时间点，尽量避开已有时间戳
- 重写时会清理旧的分析展示行，避免重复堆积
