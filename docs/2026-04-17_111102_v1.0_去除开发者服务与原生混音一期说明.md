# v1.0 去除开发者服务与原生混音一期说明

时间：2026-04-17 11:11:02 +08:00

## 历史记录总结

- 已读取 `codex_lx二开_log.db` 历史记录。
- 上一轮已完成：
  - KTV 本地变体识别与播放页入口。
  - 启动红屏的空值兼容修复。
- 上一轮未完成：
  - 真正的原生混音切轨。

## 本轮架构拆分

- 启动与打包层
  - 负责让 debug 包不再依赖 Metro / 开发者服务。
- 原生混音层
  - 新增 Android `MixerModule`，用双 ExoPlayer 做本地双轨过渡。
- JS 桥接层
  - 新增 `src/utils/nativeModules/mixer.ts`，统一管理混音状态。
- 播放控制层
  - 在 `src/plugins/player/utils.ts` 中把 `play/pause/seek/progress/volume/rate` 在 KTV 混音激活时转发到原生混音模块。
- KTV 业务层
  - 在 `src/core/ktv/index.ts` 中把当前“硬切文件”升级为“优先原生平滑切换，失败再回退单轨切换”。

## 本次修改

- Android 去除开发者测试服务依赖
  - `android/app/build.gradle`
    - 在 `react {}` 中设置 `debuggableVariants = []`，让 `debug` 也执行 JS bundling。
    - 显式加入 `androidx.media3:media3-exoplayer:1.8.0` 供原生混音模块使用。
  - `android/app/src/main/java/cn/toside/music/mobile/MainApplication.java`
    - `getUseDeveloperSupport()` 固定返回 `false`。
    - 移除 `ReactNativeFlipper` 初始化。
- 原生混音一期
  - 新增 `android/app/src/main/java/cn/toside/music/mobile/mixer/MixerModule.java`
    - 维护 `activePlayer + standbyPlayer`。
    - 支持在同一时间点预加载第二轨并做短时音量渐变。
    - 支持 `play / pause / stop / seek / position / duration / volume / rate`。
  - 新增 `android/app/src/main/java/cn/toside/music/mobile/mixer/MixerPackage.java`
  - 新增 `src/utils/nativeModules/mixer.ts`
  - 修改 `src/plugins/player/utils.ts`
    - KTV 混音激活时，把关键控制操作转发到原生混音模块。
  - 修改 `src/core/ktv/index.ts`
    - KTV 切换优先走 `MixerModule.startTransition()`。
    - 当前实现是“按当前位置 + 短时过渡”的一期版本，还没有接入节拍检测。

## 当前阻塞与边界

- `./gradlew.bat assembleDebug` 仍被项目原有 `react-native-quick-base64` 的 CMake 长路径问题阻塞，不是本轮新增逻辑导致。
- 原生混音一期当前只服务：
  - Android
  - 本地文件
  - 双轨切换
- 当前还没有实现：
  - 节拍分析与“下一拍切换”
  - 多 stem 同时混音
  - 通知栏/媒体键在混音激活态下的深度同步
  - `pitch shift` 原生实现

## 下一阶段建议

1. 把 KTV 原生混音从“当前位置过渡”升级到“拍点对齐过渡”。
2. 把原生混音状态同步进通知栏与后台 service。
3. 再做独立的 `pitch shift` 与调号识别链路。
