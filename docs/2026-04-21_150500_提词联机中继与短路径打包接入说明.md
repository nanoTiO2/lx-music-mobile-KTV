# 2026-04-21 提词联机中继与短路径打包接入说明

## 本轮目标

- 把提词控制端、展示端、被控端的可测链路补完整
- 在仓库内提供最小可用的 WebSocket relay，避免只能停留在客户端原型
- 为 Windows 中文长路径环境补一键短路径打包与安装脚本

## 已完成内容

### 1. 提词联机中继接入

- 新增 `scripts/prompt_ws_relay.js`
- 新增根目录启动脚本 `run_prompt_relay.bat`
- 新增 `package.json` 脚本 `npm run prompt:relay`

当前 relay 行为：

- 监听 `0.0.0.0:9528`
- 默认路径 `/prompt`
- 以 `sessionId` 作为房间号
- 控制端与被控端只要连接到同一 relay，并使用相同 `sessionId`，即可完成提词快照同步
- relay 为透传转发，不保存持久状态；状态主导权仍在控制端

### 2. Windows 短路径打包接入

- 新增 `build_debug_shortpath.bat`
- 新增 `install_debug_shortpath.bat`

主要用途：

- 通过 `subst` 把仓库临时映射到短盘符
- 避免中文目录和超长路径导致的 Gradle / CMake / Ninja 构建异常
- 减少每次手工切目录和找 APK 的重复操作

### 3. 测试入口串联结果

- 播放详情页 -> `提` -> `PromptControl`
- `PromptControl` -> `PromptStage`
- `PromptControl` -> `PromptReceiver`
- 设置 -> 其他 -> `开启被控端` -> `PromptReceiver`
- 歌词舞台 `LyricStage` 退出后恢复系统 UI，避免状态栏重叠

## 推荐测试顺序

1. 电脑执行 `run_prompt_relay.bat`
2. 手机安装 debug 包
3. 控制端在播放页点击 `提`
4. 被控端从设置页进入 `开启被控端`
5. 两端填写同一个 `ws://电脑IP:9528/prompt`
6. 两端填写同一个 `sessionId`
7. 在控制端导入文本并执行 `上屏 / 下一句`
8. 检查被控端是否同步显示

## 当前边界

- relay 仅解决局域网联调，不处理鉴权、断线重放、历史回补
- 如果控制端未连接 relay，被控端仍可使用本机共享会话，但不具备跨设备同步能力
- Android Debug 构建仍依赖本机 Android SDK / JDK / ADB 环境完整可用
