# 2026-04-21 快速迭代六项修复交付说明

## 本次目标

围绕“快速迭代后可直接上手机逐条测试”处理以下六类问题：

1. 歌词舞台圆盘误触与长按触发问题。
2. 歌词舞台底部菜单点按唤出与自动隐藏时序问题。
3. 在线歌曲无法执行调号分析。
4. 下载后的文件需要可直接播放，并默认映射到本地音乐目录。
5. 从不同歌单点击播放后，后续播放顺序错误沿用旧列表。
6. 浏览本地其他文件夹时会打断当前播放，且后续顺序被错误切换。

- 歌词舞台交互层：`src/screens/LyricStage/index.tsx`
- 播放上下文层：`src/core/player/player.ts`

- 本地文件夹浏览层：`src/screens/Home/Views/LocalMusic/index.tsx`
- 下载与本地映射层：`src/core/download/index.ts`、`src/screens/Home/Views/Download/index.js`
- 调号分析层：`src/screens/PlayDetail/components/KeyInfoBtn.tsx`

## 已完成修复

### 1. 歌词舞台圆盘

- 取消长按圆盘触发链路，改为显式按钮触发。
- 圆盘按钮放到辉光灯按钮右侧，避免鼠标移动误触。
- 圆盘提示文案改为“按键呼出圆盘”。
- 圆盘和面板字体改为按屏幕宽度自适应，避免大字体场景下文字超边界。

### 2. 歌词舞台菜单显示逻辑

- 点按舞台即可唤出控制菜单，不再依赖滑动。
- 菜单和圆盘空闲隐藏时间统一从 1 秒调整为 2 秒。
- 点击主菜单、二级设置、滑杆等操作后都会续期隐藏计时。

### 3. 辉光灯按钮

- 辉光灯改为一个循环按钮。
- 点击顺序为：关 -> 青 -> 金 -> 玫 -> 关。
- 保留辉光风格二级设置，主按钮负责开关与颜色轮换。

### 4. 在线调号分析

- 当前播放在线歌曲时，分析前会先下载到缓存目录，再调用原有分析链路。
- 本地歌曲仍继续走原来的本地分析与 LRC/缓存写回逻辑。
- 在线歌曲分析完成后会缓存结果，避免重复分析。

### 5. 下载链路

- 下载目录会自动加入本地音乐导入目录映射。
- 下载页完成态任务增加“播放”按钮，可直接播放下载后的文件。
- 播放前会补一次下载列表入库，避免刚下载完成时列表尚未同步。

### 6. 播放列表与本地浏览上下文

- `TEMP` 列表不再只按固定 `temp` 标识判定上下文，而是带上真实临时列表来源标识，切换不同在线歌单时会正确清空旧播放历史。
- 浏览本地其他文件夹前，如果当前歌曲来自本地列表且不属于即将浏览的文件夹，会先冻结当前播放上下文快照。
- 因此：浏览 B 文件夹不会打断正在播放的 A；只有真正点击 B 中歌曲时，后续顺序才切换到 B。
