## Android 打包过程中的实际问题与处理

- 原工程目录位于含中文路径的 `C:\Users\Administrator\Desktop\lx二开\...`
- Android Gradle Plugin 会阻止非 ASCII 路径；已通过 `android.overridePathCheck=true` 放开
- React Native 自动链接读取 Node CLI 输出时受 Windows 默认编码影响；已通过 `-Dfile.encoding=UTF-8` 修正
- `react-native-quick-base64` 的 CMake 构建在原长路径下触发对象文件路径过长，导致 `build.ninja` 持续 dirty
- 最终采用短 ASCII 构建工位 `F:\lxm` 解决打包问题
