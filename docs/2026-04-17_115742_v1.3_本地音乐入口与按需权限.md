# v1.3 本地音乐入口与按需权限

## 需求摘要

- 启动时不再主动申请存储权限，改为功能使用时再申请。
- 左侧菜单新增“本地音乐”，位置在“下载”下方、“设置”上方。
- 进入后显示“添加本地文件夹（含子目录递归）”，手工触发扫描。
- 扫描后直接复用现有播放页、歌词页和列表播放能力。
- 最终需要完成 Android 构建并通过 `adb` 安装到已连接设备。

## 架构拆分

- 初始化模块：`src/core/init/index.ts`
  - 职责：应用启动时仅检查权限状态，不主动弹窗。
- 导航模块：`src/config/constant.ts`、`src/screens/Home/Horizontal/Main.tsx`、`src/screens/Home/Vertical/Main.tsx`
  - 职责：新增 `nav_local` 菜单和页面路由。
- 本地扫描模块：`src/utils/localMediaMetadata.ts`
  - 职责：递归扫描目标目录，过滤显示用伴奏/鼓点等衍生文件，产出可导入曲目。
- 歌单导入模块：`src/screens/Home/Views/Mylist/MyList/listAction.ts`
  - 职责：将递归扫描结果覆盖写入 `LOCAL_MUSIC` 虚拟列表，并复用已有元数据刷新流程。
- 本地音乐页：`src/screens/Home/Views/LocalMusic/index.tsx`
  - 职责：选择目录、触发扫描、显示当前目录、复用 `MusicList` 展示与播放。
- 列表状态模块：`src/store/list/*`
  - 职责：保留 `LOCAL_MUSIC` 作为内置虚拟列表 id，但不混入用户歌单抽屉。

## 数据流

1. 用户点击左侧 `本地音乐`。
2. 页面调用目录选择器，用户手工选择一个根目录。
3. 软件记录 `list.importMusicDir`，随后递归扫描该目录及子目录。
4. 扫描结果写入 `LOCAL_MUSIC` 列表 id。
5. `MusicList` 直接读取当前激活列表并复用现有播放页面。

## 本次关键改动

- 启动时禁用主动权限申请，避免一进软件就弹授权。
- 新增 `本地音乐` 菜单入口，并修正为现有字体图标中的 `add_folder`。
- 新增独立的本地音乐页，支持：
  - 添加本地文件夹（含子目录递归）
  - 手工重新扫描
  - 展示当前目录
  - 使用原有列表与播放页
- 将 `LOCAL_MUSIC` 从 `allList` 聚合中移出，避免显示在用户歌单抽屉里。

## 风险与说明

- 当前本地音乐页复用 `MusicList`，因此顶部交互仍沿用现有歌单页样式，这是刻意保持最小改动。
- 项目已有若干历史 TypeScript 报错，与本次改动无关；本次以 `eslint` 和 Android 构建结果作为交付校验主线。
- Windows 中文路径下 Android 原生构建存在不稳定性，建议复制到 ASCII 路径后执行 Gradle。
