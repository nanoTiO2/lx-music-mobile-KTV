# 2026-04-21 播放器返回布局与循环分析二期修复

## 本轮目标

1. 修复歌词舞台返回后播放页顶部空间未恢复的问题。
2. 修复单曲循环实际停住并短暂进入“新曲目/获取链接”状态的问题。
3. 延伸曲调分析二级展示，补充独立和弦稿产物。
4. 阻断持续返回直接退出到桌面的行为。

## 架构拆分

- 系统 UI 恢复链：
  - `src/screens/LyricStage/index.tsx`
  - `src/screens/PlayDetail/index.tsx`

  - `src/utils/statusBar.ts`

  - `src/plugins/player/utils.ts`

  - `src/utils/hooks/useBackHandler.ts`

## 关键修改

### 1. 歌词舞台退出后的顶部布局恢复

- 新增 `src/utils/statusBar.ts`，统一计算安全的状态栏高度。
- `PlayDetail` 在 `didAppear` 时重复同步状态栏高度，并重新应用 `statusBar/navigationBar/layout` 配置。
- `LyricStage` 退出时恢复为应用本身使用的透明状态栏模型，不再切回 `translucent=false`。
- `SizeView` 改为使用安全状态栏高度计算，减少沉浸式切换后 `StatusBar.currentHeight` 暂时异常导致的顶栏重叠。

### 2. 单曲循环

- 修正 `setLoop()` 的逻辑错误，`singleLoop` 现在使用 TrackPlayer 的 `RepeatMode.Track`。
- 播放器初始化和播放模式切换时都同步底层 repeat mode。
- 删除 service 层“队列结束后补救式 reset+play”的单曲循环逻辑，避免停顿、空队列、默认占位曲目和“新曲目/获取链接”状态闪现。

### 3. 曲调分析二级展示

- 分析结果额外落盘到：
  - `*.lx-profile-detail.json`
  - `*.lx-chords.lrc`
- 和弦稿优先按歌词时间轴对齐；无歌词时退化为仅按分析分段输出。
- 播放页调号按钮已提供二级分析入口，展示：
  - 大调/小调与主音
  - 最高音 / 常见高音
  - 最低音 / 常见低音
  - 拍号推断
  - 和弦分段摘要

### 4. 返回键阻断

- `useBackHandler` 改为使用订阅对象移除监听，避免旧式 `removeEventListener` 带来的失效风险。
- 根页面现有“只回菜单、不退桌面”的逻辑继续保留。

## 本轮仍需你重点实机确认

1. 从播放页进入歌词舞台后返回，顶部是否已经重新留出状态栏空间。
2. 单曲循环是否实现无停顿原地重播。
3. 返回手势连续触发时，是否只停留在应用内，不再退回桌面。
4. 二级分析弹层和独立 `.lx-chords.lrc` 文件的内容是否符合你的展示预期。
