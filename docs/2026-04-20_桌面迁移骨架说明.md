# 桌面迁移骨架说明

时间：2026-04-20 09:00 +08:00

## 目标

这次不是直接做桌面端，而是先把未来会反复修改的部分抽成：

- 共享协议层
- 桌面适配接口层
- 模块迁移计划层

这样以后迁移到 Electron 或其他桌面壳时，不需要重新从移动端源码里手工拆协议。

## 新增目录

### `src/shared/`

#### `src/shared/musicProfileTag.ts`

- 抽出了 `lx_music_profile` 的 LRC 标签协议
- 提供：
- profile 数据结构
- 从 LRC 解码 profile
- 把 profile 写回 LRC 头部

#### `src/shared/lyricStagePresets.ts`

- 抽出了歌词舞台的公共预设
- 包括：
- 字体列表
- 配色主题
- 显示模式顺序
- 镜像顺序
- 旋转顺序

### `src/desktopPort/`

#### `src/desktopPort/contracts.ts`

- 定义桌面端适配接口：
- `DesktopAudioAdapter`
- `DesktopWindowAdapter`
- `DesktopFileAdapter`

这些接口的作用是把桌面平台差异收口，避免桌面版直接复制 Android 原生逻辑。

#### `src/desktopPort/modulePlan.ts`

- 定义当前各模块迁移到桌面的优先级：
- `ready_now`
- `needs_adapter`
- `rewrite_required`

目前判断：

- `local_offline_import_cache`：可以直接迁移
- `manual_profile_analysis`：需要桌面音频分析器
- `lyric_stage_projection`：需要桌面窗口适配
- `seek_sync`：需要桌面播放器回调适配
- `mixer_pitch_seek`：必须重写底层实现

## 当前已经接入到移动端的共享层

### `src/core/musicProfile.ts`

已经改为复用 `src/shared/musicProfileTag.ts`，因此：

- LRC 分析标签协议不再只存在于移动端文件里
- 未来桌面端读取 / 写回同一套标签时可直接复用

### `src/screens/LyricStage/index.tsx`

已经改为复用 `src/shared/lyricStagePresets.ts`，因此：

- 歌词舞台配色、字体、模式预设不再硬编码在单页面里
- 桌面端歌词舞台 UI 可以直接继承这套预设

## 后续桌面迁移建议顺序

### 第一阶段

- 迁移 `src/shared/musicProfileTag.ts`
- 迁移本地离线导入缓存逻辑
- 完成桌面文件选择和本地路径管理

### 第二阶段

- 迁移歌词舞台页面 UI
- 接入 `DesktopWindowAdapter`
- 实现全屏、置顶、背景色控制

### 第三阶段

- 接入桌面音频引擎
- 实现：
- seek
- rate
- pitch
- analyzeMusicProfile

### 第四阶段

- 对接歌词同步
- 回归测试拖动进度条、歌词对齐、LRC 写回

## 结论

这次新增的不是“桌面版功能”，而是“桌面版施工骨架”。

价值在于：

- 以后继续二开时，不会把共享协议继续埋在移动端页面内部
- 未来桌面迁移时，可以先接 adapter，再接 UI，再接音频底层
- 后续多人协作时，模块边界更清楚
