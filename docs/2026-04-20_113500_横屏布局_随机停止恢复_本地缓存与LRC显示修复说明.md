# 2026-04-20 横屏布局、随机/停止恢复、本地缓存与 LRC 显示修复说明

## 本轮目标

- 修复单曲循环重播时进度条与时间不同步、结尾出现短暂空白新曲目的问题
- 恢复随机播放与播完停止模式
- 本地歌曲评论按钮改为灰色禁用态
- 横屏播放页左上角按钮与 Header 重叠问题缓解
- 横屏捏合/重渲染时，本地音乐页尽量走缓存，避免重复扫描并重复导入
- 强化 LRC 调号/拍速/节拍/分析时间的可显示写回

## 关键修复

### 1. 单曲循环

- `playerEnded` 在 `singleLoop` 模式下先 `seekTo(0)`，再重新播放
- 目标是让进度条、当前时间和实际音频一起从头开始，不再先空跑 1~2 秒

### 2. 播放模式恢复

- 横屏和竖屏播放模式按钮重新恢复 5 态循环：
  - 列表循环
  - 随机播放
  - 顺序播放
  - 单曲循环
  - 播完停止
- 同时保留单曲循环的应用层重播兜底

### 3. 本地评论按钮

- 播放页评论按钮在本地歌曲下直接变灰并禁用
- 评论详情页继续保持灰色提醒态

### 4. 横屏布局

- 左侧竖排功能按钮整体右移、下移
- 左侧内容区域同步增加左侧预留，减少与返回按钮、计时按钮重叠

### 5. 本地音乐缓存

- 本地目录页新增目录签名缓存与分组缓存
- 同一批导入目录在纯重渲染场景下优先复用缓存结果
- 避免横屏捏合或无关刷新时再次扫描目录、再次导入同一文件集合

### 6. LRC 显示写回

- 保留顶部元标签：
  - `key`
  - `bpm`
  - `beat`
  - `analysis_time`
  - `lx_music_profile`
- 额外写入 4 行真正可见的时间轴歌词：
  - `调号：...`
  - `拍速：...`
  - `节拍：...`
  - `分析时间：...`
- 时间优先分配在 `0s~1s` 内，并自动避开已有起始歌词时间戳
