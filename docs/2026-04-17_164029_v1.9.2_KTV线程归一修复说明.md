# v1.9.2 KTV线程归一修复说明

- 时间: 2026-04-17T16:40:29.565+08:00
- 目标: 修复播放中 KTV 切换导致的卡死、wrong-thread 异常、RN Promise Long 类型崩溃。

## 架构调整

- 为 `MixerModule` 增加独立 `HandlerThread`，统一承载 `ExoPlayer` 创建、播放、暂停、seek、轮询、交叉渐隐、释放。
- `watchSwitchPoint` 与 `runCrossfade` 不再跑在主线程，避免 Media3 `Player is accessed on the wrong thread`。
- Promise 回调统一回到 React Native native modules queue。

## 关键修复

- `getPosition()` / `getDuration()` 改为返回 `double`，避免 `Cannot convert argument of type class java.lang.Long`。
- 保留现有拍点切换 + 交叉渐隐机制，不继续引入新的预热策略，先确保稳定性。

- Android Debug APK 已重新构建成功。
- 已安装到测试设备 `f158b99c`。
- 当前等待用户复现一次 KTV 切换，进一步确认是否彻底消除卡死与残音。
