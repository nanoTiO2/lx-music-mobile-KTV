## 架构拆分

### 1. UI 入口层

- 播放页 `KtvBtn` 负责展示音轨切换按钮与弹窗
- 进入歌曲后预热 beat grid 分析，减少首次切换等待

### 2. KTV 业务层

- 扫描本地同目录音频文件
- 识别 `original / accompaniment / bass / drums / main` 变体
- 切换时保留原歌曲元数据、歌词、封面与当前播放位置

### 3. Beat 分析层

- 通过原生 `MixerModule.analyzeBeatGrid()` 对音频前段做简单节拍估计
- JS 层将结果缓存到内存与磁盘
- 切换时根据当前播放位置推算“下一拍”的切换时间

### 4. 原生混音层

- Android 原生 `MixerModule` 使用双 `ExoPlayer`
- 当前轨与目标轨在下一拍切入，并执行短时 crossfade
- `play / pause / seek / rate / volume / duration / position` 已接到 mixer 通道
