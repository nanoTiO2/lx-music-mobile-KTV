# v0.4 下载闭环与播放页入口说明

## 本次完成

- 播放详情页补了“下载/保存到本地”入口
  - 竖屏 `MoreBtn`
  - 横屏 `MoreBtn`
- 下载页补了任务操作
  - 开始
  - 暂停
  - 重试
  - 删除
  - 清理已完成
- 下载任务播放链路改成“本地优先”
  - 已完成任务优先直接使用落盘文件路径
  - 不再强制再次请求在线 URL
- 下载完成后会尝试把音频文件加入试听列表
  - 先插入本地歌曲
  - 再读取元数据补全歌名、歌手、专辑、时长

## 关键文件

- `src/core/download/index.ts`
- `src/core/music/download.ts`
- `src/screens/Home/Views/Download/index.js`
- `src/screens/PlayDetail/Vertical/Player/components/MoreBtn/DownloadBtn.tsx`
- `src/screens/PlayDetail/Horizontal/MoreBtn/DownloadBtn.tsx`

## 当前效果

- 在线列表长按可以下载
- 播放详情页当前歌曲也可以直接下载
- 下载页现在是可操作任务面板，不再只是展示
- 已下载资源后续会优先按本地文件参与播放链路

## 仍保留的限制

- 还没有专门的“下载列表播放页”
- 自动导入目前接到试听列表，不是独立“本地下载”歌单
- 暂未做断点续传、并发数控制、任务持久恢复中的运行态恢复
- 还没有把伴奏/原唱切源与下载产物做逻辑聚合

## 收尾结论

到这一阶段，下载主线已经从“占位设计”推进到“应用内可操作、可执行、可复用本地文件”的闭环版本，可以作为后续本地聚合导入与 KTV 切源的基础。
