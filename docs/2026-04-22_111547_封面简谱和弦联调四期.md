1. 封面

## 3. 架构拆分

### 3.1 和弦

此前 beat-grid 修正主要依赖“已有和弦段标签”，仍容易被前一轮切段偏差带歪。本轮新增一层：

- 按拍重新聚合 `pitchClassEnergy`
- 在每拍上直接重新打分简单三和弦候选
- 再与已有 segment 候选一起进入后续 DP 与小节压缩

这意味着：

- 旧链路负责给出初步和弦段
- 新增链路负责用原始 chroma 在 beat 级做二次校正

### 3.2 简谱

此前简谱子拍统计里，每一帧都用了固定权重 `0.1`，没有使用原生 pitch frame 的实际能量，导致很多“有效音高没抓到”或被弱噪声覆盖。

本轮改为：

- 原生返回 `pitchTrack.weight`
- 前端按真实权重聚合每个子拍的音高
- 简谱页把空拍记号显示为 `·`，增强可读性

### 3.3 封面

此前下载封面虽已支持 sidecar，但本地 `file://` 路径遇到空格/中文时，存在 URI 没标准化的问题，容易导致手机上看起来“封面又没了”。

本轮统一：

- 对 sidecar 封面路径和本地读取到的 `file://` / `content://` 路径做 `encodeURI`
- 在导入下载列表、同步下载列表、保存下载封面三个阶段统一标准化

## 4. 代码改动

### 4.1 原生和弦二次候选

文件：

- `android/app/src/main/java/cn/toside/music/mobile/mixer/MixerModule.java`

新增：

- `buildBeatPitchClassEnergy(...)`
- `addBeatPitchCandidates(...)`

改动点：

1. `alignChordSegmentsToBeatGrid(...)` 新增 `segmentPitchClasses` 入参
2. `buildBeatCandidateScores(...)` 不再只看已有和弦段标签
3. 对每一拍重新聚合原始 chroma
4. 仅对简单三和弦：
   - 大三
   - 小三
   - 减三
   做重新打分
5. 再叠加强拍/调内位置奖励，作为新的 beat 候选分数来源

这样可以避免：

- 第一轮分段一旦错，后面全都跟着错
- 一整段被单一和弦“拖平”

### 4.2 简谱改为按真实权重聚合

文件：

- `src/screens/PlayDetail/components/KeyInfoBtn.tsx`
- `src/shared/musicProfileTag.ts`
- `src/utils/nativeModules/mixer.ts`

改动点：

1. 原生 `pitchTrack` 回传 `weight`
2. TS 类型同步补上 `weight?: number`
3. `buildPitchSlots(...)` 由固定权重改为优先使用 frame weight
4. 简谱行里的 `0` 改显示为 `·`

结果：

- 子拍主音的选取更接近真实能量峰值
- 空拍、弱拍、延音在纯文本里更容易读

### 4.3 下载封面本地 URI 标准化

文件：

- `src/core/download/index.ts`

新增：

- `normalizeLocalUri(...)`

改动点：

1. sidecar 封面路径改为统一走 `encodeURI(file://...)`
2. `readPic(...)` 返回的本地 `file://` / `content://` 也统一标准化
3. 保存封面后立即把 `task.metadata.musicInfo.meta.picUrl` 写成标准化本地 URI

作用：

- 修正中文路径、空格路径下的封面显示不稳定问题
