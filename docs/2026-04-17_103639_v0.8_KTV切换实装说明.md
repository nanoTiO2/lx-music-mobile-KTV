# v0.8 KTV切换实装说明

时间：2026-04-17 10:36:39 +08:00

## 本次增量

- 播放页新增 `KTV` 入口，横屏与竖屏都可打开音轨选择面板。
- 本地歌曲会扫描当前目录的拆轨文件，识别 `主文件 / 原唱 / 伴奏 / 低音 / 鼓组`。
- 切换 KTV 音轨时复用当前单轨播放器，尽量保留当前播放进度。
- KTV 变体切换后，歌词与封面优先继续读取主文件或原唱文件，避免切到伴奏后丢歌词。

## 架构拆分

- `src/utils/ktv.js`
  - 负责拆轨命名识别与分组。
- `src/core/ktv/index.ts`
  - 负责目录扫描、生成 KTV 选项、执行播放切换。
- `src/core/music/local.ts`
  - 负责本地歌词与封面回退到主文件元数据。
- `src/screens/PlayDetail/components/KtvBtn.tsx`
  - 负责播放页弹层与用户交互。

## 关键行为

- 当前版本是单轨切换，不是双轨混音。
- 切换时会替换当前播放资源，并尝试保持原播放时间点。
- 如果歌曲当前处于暂停态，切换后会补一次暂停，尽量维持原状态。

## 仍然存在的边界

- 还没有做双轨实时混音、原唱/伴奏音量独立控制。
- 当前 KTV 识别依赖文件命名规则；极端命名仍可能识别不到。
- 切换暂停态时通过补暂停维持状态，仍不是底层播放器原生无缝切轨。
