# v1.91 播放中拖动卡顿与歌词对齐修复说明

时间：2026-04-19 23:34 +08:00

## 本轮目标

- 继续修复本地播放过程中，底部进度条拖动后音频切换卡顿的问题。
- 继续压缩拖动后歌词与音频容易不同步的问题。
- 保持现有 `v1.91` 版本号不变，重新构建 Android 安装包交付测试。

## 架构判断

- JS 层进度条事件已经会落到统一的 `setCurrentTime()`。
- 本地播放在变调、变速或本地 mixer 激活后，最终会走 Android 原生 `MixerModule.seekTo()`。
- 之前 JS 侧已经做过轮询抑制和 seek 后位置回读，但原生 `seekTo()` 仍然是直接对 ExoPlayer 生硬调用，缺少 seek 前后的状态收口，因此播放中拖动仍可能卡顿、回跳、歌词错位。

## 本轮修复

- 修改 `android/app/src/main/java/cn/toside/music/mobile/mixer/MixerModule.java`
- `seekTo()` 现在会在 seek 前清理旧的切换监听器和淡入淡出任务，避免旧状态继续抢写播放器。
- seek 前会先暂停 active / standby 播放器，seek 后重新应用变速、变调和音量。
- 对跨轨过渡场景增加 seek 后状态整理：如果用户已经拖过切换点，则直接提升 standby 为 active，避免继续保留过时切换状态。
- 增加极短的稳定延迟后再恢复播放并返回成功，减少 ExoPlayer 仍在重定位时 JS 侧立即读取位置造成的抖动。
