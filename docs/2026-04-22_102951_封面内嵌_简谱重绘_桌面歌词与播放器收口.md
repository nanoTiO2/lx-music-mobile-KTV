## 3. 关键实现说明

### 3.1 下载封面自动写回音频标签

在 `src/core/download/index.ts` 中新增了 `buildCoverPath()` 与 `saveTaskCover()`：

- 下载任务完成后：
  - 优先使用任务元数据里的 `picUrl`；
  - 若缺失，则回退调用在线源的 `getPicUrl()`；
  - 封面下载到 `xxx.cover.jpg` 后，立即调用 `writePic(task.metadata.filePath, coverPath)` 写回音频标签。
- 对历史歌曲兼容：
  - `importDownloadedMusicToDownloadList()` 和 `syncDownloadedList()` 现在会：
    - 先读音频自身内嵌封面；
    - 若没有，再看 `.cover.jpg` 是否存在；
    - 若存在则把 sidecar 封面挂到界面上，并异步尝试再次写回音频标签。

这样处理后，封面既能即时显示，也能逐步把历史下载歌曲修复成“音频自身带封面”的状态。

### 3.2 简谱改成纯文本风格

在 `KeyInfoBtn.tsx` 内新增：

- `buildJianpuMeasureText(block)`
- `buildJianpuLyricText(block)`
- `SheetBeatCell`
- 新版 `SheetBlock`
- 基于拍点的小节构造流程：
  - `buildBeatBoundaries()`
  - `buildSheetAnchors()`
  - `buildBeatLyricFragments()`
  - `buildSheetBlocks()`

新的展示逻辑：

- 和弦 / 级数：
  - 仍然基于“小节 x 拍子”的二维格子展示。
  - 每个拍格可显示对应和弦或级数，以及该拍附近的歌词碎片。
- 简谱：
  - 不再展示成拥挤卡片。
  - 改为近似 `txt` 乐谱形式，例如：

```txt
01|1234 | 1  2  3  5 |
   | 你  好  世  界 |
```

这样更接近用户想要的“纯文本可读简谱”。

### 3.3 级数逻辑继续按大调体系输出

`degreeTextForChord()` 已改为直接基于主音与和弦根音间隔推导级数，统一走大调参考体系：

- 大三和弦输出 `1 / 4 / 5` 这类形式；
- 小和弦输出 `2m / 3m / 6m`；
- 减和弦输出 `7dim`。

这部分仍然依赖当前本地和弦分析结果本身的正确率；本轮主要解决展示体系与可读性，不把复杂说明文案混进结果页。

### 3.4 升降调与速率按钮高对比化

`KeyInfoBtn.tsx` 中 `quickBtn` 样式已改为深色按钮：

- 背景色：`#274863`
- 边框色：`#183044`

在浅色分析页里不再出现按钮和背景对比过低的问题。

### 3.5 桌面歌词背景透明度

本轮补齐了桌面歌词背景透明度链路：

- 设置类型中新增 `desktopLyric.style.backgroundOpacity`
- 设置页新增 `BackgroundOpacity` 控件
- JS 层 `showDesktopLyricView()` 新增 `backgroundOpacity`
- 原生桥接新增 `backgroundAlpha` / `setBackgroundAlpha`

该参数只作用于桌面歌词，不影响歌词舞台页面。
