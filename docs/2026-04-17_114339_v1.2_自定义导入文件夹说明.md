# 2026-04-17 v1.2 自定义导入文件夹说明

## 本次目标

- 为本地歌曲导入补上“自定义导入文件夹”能力
- 让用户不再每次从存储根目录重新选择导入目录
- 保持现有“我的列表 -> 添加本地歌曲”导入链路不变

## 架构拆分

### 1. 设置层

- 在“设置 -> 列表”中新增默认导入文件夹设置项
- 支持选择目录、查看当前目录、清空目录

### 2. 目录选择层

- 扩展 `ChoosePath`，支持传入 `initialDir`
- 选择目录时优先打开保存过的默认导入文件夹

### 3. 导入业务层

- 继续复用现有 `MyList -> ListImportExport -> handleImportMediaFile()` 链路
- 当用户从“添加本地歌曲”重新选择了其他目录时，自动更新默认导入目录

## 行为变化

### 1. 新增设置项

位置：

- 设置 -> 列表 -> 本地歌曲默认导入文件夹

能力：

- 选择文件夹
- 清空已保存目录
- 显示当前已保存路径

### 2. 导入入口默认定位

原来的“我的列表 -> 列表菜单 -> 添加本地歌曲”现在会：

- 优先打开 `list.importMusicDir`
- 如果用户本次改选了其他目录，则自动把该目录保存为新的默认导入目录

### 3. 持久化方式

- 新增设置键：`list.importMusicDir`
- 默认值为空字符串

## 验证结果

- 新增改动的 ESLint 检查通过
- 三份语言 JSON 语法校验通过
- 全量 `tsc --noEmit` 仍有仓库原有旧报错，但本次新增的导入目录相关类型错误已消除

## 已知边界

### 1. 仍然是单层目录扫描

- 当前 `scanAudioFiles()` 只扫描所选目录本身，不递归子目录

### 2. 仍复用现有权限模型

- 目录导入依然走当前应用内目录浏览器与已有存储权限链路
- 没有在本轮额外重构 SAF 持久目录授权逻辑

### 3. 历史旧问题仍在

设备日志里的旧初始化问题：

```text
init storage and download dir failed: Cannot read property 'isDirectory' of null
```

该问题与本次新增设置项不是同一问题，后续可单独收口。

## 结论

本次已经补齐“自定义导入文件夹”的基础能力，并接到了现有本地歌曲导入入口上。当前结果是：

- 用户可以在设置中预先配置默认导入目录
- 导入本地歌曲时会直接打开该目录
- 重新选目录后会自动记忆

这已经满足“支持自定义导入文件夹”的最小可用版本。
