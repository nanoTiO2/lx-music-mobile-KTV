# 舞台提词控制端本地联调一期说明

时间：2026-04-21 12:35 +08:00

## 本轮目标

在上一轮“展示端骨架”基础上，把当前仓推进到可本地联调的控制端原型：

- 增加共享提词会话层
- 增加控制端页面
- 打通控制端与展示端对同一份状态的联动
- 保持后续切换 WebSocket / WebRTC / 蓝牙时不重写页面层

## 本轮新增

### 1. 共享会话层

新增：

- `src/shared/prompt/session.ts`

当前职责：

- 在当前仓维护单例 `PromptSessionState`
- 对外提供 `usePromptSession`
- 对外提供 `dispatchPromptSessionCommand`
- 支持 `loadPromptSessionFromText`
- 支持 `loadPromptSessionFromPaste`
- 支持 `resetPromptSessionToDemo`

当前定位是“本地 loopback 会话层”，不是最终联网同步层。

### 2. 控制端原型页

新增：

- `src/screens/PromptControl/index.tsx`

当前能力：

- 载入 demo 文档
- 导入本地 `.lrc` `.srt` `.ass` `.txt`
- 从剪贴板导入文本
- 手动粘贴文本并解析为提词文档
- 顺序点按 / JK 接力控制
- 预览当前句 / 下一句 / 后一句
- 随时打开展示端页验证联动

### 3. 展示端改为消费共享会话

已更新：

- `src/screens/PromptStage/index.tsx`

当前变化：

- 不再只消费本页 demo 状态
- 改为订阅共享 `PromptSessionState`
- 控制端与展示端在同一应用栈内可直接联动

### 4. 入口调整

已更新：

- `src/screens/PlayDetail/components/PromptStageBtn.tsx`
- `src/navigation/screenNames.ts`
- `src/navigation/navigation.ts`
- `src/navigation/registerScreens.tsx`
- `src/screens/index.ts`

当前入口策略：

- 播放详情页“提”按钮先进入 `PromptControl`
- `PromptControl` 内可继续打开 `PromptStage`

## 这次没有做的内容

以下仍留在下一阶段：

- 真正的局域网 WebSocket 同步
- WebRTC DataChannel 会话同步
- 蓝牙链路适配
- 多展示端状态广播
- 控制端与展示端跨设备发现
- 会话持久化与断线恢复

## 建议的下一阶段顺序

1. 给 `session.ts` 抽出 transport adapter 接口
2. 先补局域网 WebSocket 会话同步
3. 再补控制端设备列表与会话创建 UI
4. 最后再接 WebRTC / 蓝牙
