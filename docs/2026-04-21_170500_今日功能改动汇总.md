## 4. 今日重要实现细节

### 4.1 调号显示体系统一

用户要求尽量用大调体系展示主调，因此在显示层做了统一：

- 小调分析结果在外部展示时转换到对应大调体系
- 示例：如果内部分析倾向于某小调，外显调号可落到对应大调主音体系

相关函数位于：

- `src/screens/PlayDetail/components/KeyInfoBtn.tsx`

包括：

- `getMajorSystemDisplayKeyLabel`
- `getMajorSystemTonicName`

### 4.2 LRC 写回极简化

为避免污染原歌词，写回规则收敛为仅保留：

- `[key:...]`
- `[bpm:...]`
- `[lx_music_profile:base64-json]`
- 展示行中的“调号”“拍速”

这确保：

- 外部播放器和用户肉眼阅读更干净
- 内部仍可恢复分析缓存

### 4.3 本地缓存优先级调整

在今天后半段，为解决“全曲分析完成后下次又像没保存”的反馈，调整了缓存读取顺序：

旧逻辑：

- 优先从 `.lrc` 读内嵌 profile
- 再读 `.lx-profile.json`

新逻辑：

- 优先从 `.lx-profile.json` 读取完整缓存
- 再回退到 `.lrc` 内嵌信息

这样做的原因：

- `.lx-profile.json` 保存内容更完整
- 避免旧 `.lrc` 里的历史嵌入结果覆盖更新后的全曲分析结果

### 4.4 和弦展示的“简单化”策略

用户明确反馈复杂和弦显示过于混乱，因此展示层做了“收敛”：

- 显示时优先保留根音与大小性质
- 尽量简化为 `C / D / F / G / Am / Em` 这类形式
- 不再优先暴露大量 `add9 / sus4 / maj7 / 9 / 11` 之类复杂后缀

说明：

- 这是展示层收敛，不等于底层分析完全不区分复杂和弦
- 后续如需专业模式，可再做“简洁模式 / 专业模式”双轨

### 4.5 简谱模式

为了满足“txt 展示简谱”的需求，今日补入 `pitchTrack`：

- 原生分析阶段记录采样音高轨迹
- 返回到前端后按歌词行分配音高
- 基于调号主音换算成简谱级数
- 用文本方式展示为数字旋律行

当前实现目标是：

- 先提供可读的 txt 简谱
- 后续再视测试情况继续提高准确率与排版精细度
