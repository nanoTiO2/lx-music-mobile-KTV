# 定制模块部署与合并清单

时间：2026-04-20 00:00 +08:00

## A. 主线升级时的执行清单

### 0. 升级前准备

- 备份当前二开目录
- 备份两个 SQLite 日志数据库
- 阅读 `src/customModules/moduleManifest.ts`
- 阅读最近的 `docs/时间戳_说明.md`

### 1. 准备上游新版本

- 单独下载或拉取新的开源主线代码
- 不要在你的二开目录上直接覆盖
- 形成：
- `upstream_new/`
- `custom_current/`

### 2. 先做结构对比

重点先看这些目录是否有重构：

- `src/plugins/player/`
- `src/core/init/player/`
- `src/core/music/`
- `src/screens/PlayDetail/`
- `src/screens/LyricStage/`
- `android/app/src/main/java/cn/toside/music/mobile/`

### 3. 按模块迁移

按下面顺序迁移最稳：

1. `local_offline_import_cache`
2. `permission_compat`
3. `manual_profile_analysis`
4. `lyric_stage_projection`
5. `seek_sync`
6. `mixer_pitch_seek`

### 4. 每迁移一个模块都要做的事

- 检查 `moduleManifest.ts` 中列出的文件是否全部处理
- 检查设置项有没有丢失
- 检查对应按钮入口是否还在
- 检查语言文案是否补齐
- 检查该模块是否仍然能单独工作

### 5. 主线升级后的最低回归测试

- 本地目录导入
- 本地歌曲播放
- 播放中拖动底部进度条
- 变调 / 变速
- 手动调号 / 节拍分析
- 歌词舞台进入 / 退出 / 横竖屏 / 纯黑模式
- 安卓 7 或低配机权限流程

## C. 电脑端迁移清单

### 第一阶段：先迁移纯业务逻辑

- 本地导入缓存
- 本地歌词与元数据离线优先
- 手动分析缓存协议
- `lx_music_profile` LRC 标签读写

### 第二阶段：迁移 UI

- 歌词舞台页面
- 配色切换
- 菜单显示/隐藏策略
- 字体选择
- 纯黑投影模式

### 第三阶段：替换平台能力

- Android 权限逻辑 -> 桌面文件选择
- Android 状态栏 / 全屏 -> 桌面窗口全屏 API
- Android MixerModule -> 桌面音频引擎适配层

### 第四阶段：桌面最低回归测试

- 本地文件导入后是否复用缓存
- 歌词舞台是否支持投影展示
- 拖动进度条后歌词是否能回正
- 分析结果是否能写回 LRC 并再次读取
- 变调 / 变速是否仍可用

## D. 后续新增功能的维护要求

- 新功能先登记到 `src/customModules/moduleManifest.ts`
- 每个重要版本补一份 `docs/时间戳_说明.md`
- 每次构建交付更新 SQLite 日志
- 不再允许只改代码不记模块归属
